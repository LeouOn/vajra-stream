## Documentation Standards

### Markdown Files
- Use clear headings (##, ###)
- Include table of contents for long docs
- Use checklists [ ] for tasks
- Add emojis for visual clarity (optional)
- Keep lines under 100 characters when possible

### Code Documentation
- Docstrings for all classes and functions
- Inline comments for complex logic
- Type hints where applicable
- Examples in docstrings

### Knowledge Base (JSON)
- Use clear, consistent naming
- Include `_comment` and `_todo` fields
- Document data sources
- Validate JSON syntax

## Contributing Documentation

When you create or update documentation:

1. **Be Clear** - Write for someone unfamiliar with the topic
2. **Be Accurate** - Cite sources, especially for healing protocols
3. **Be Safe** - Always include safety information and contraindications
4. **Be Respectful** - Honor traditional sources and lineages
5. **Be Helpful** - Include examples, checklists, practical guidance
