# Compassionate Blessing Slideshow System

## Overview

The Blessing Slideshow is an automated system that cycles through photographs of beings, overlaying sacred mantras and positive intentions for rapid blessing transmission. It functions like a **high-speed digital prayer wheel for visual witness samples**.

### Inspired By

- **Prayer Wheel Technology** - Rapid repetition of mantras for blessing
- **Radionics Witness Principle** - Photographs as energetic connections
- **Subliminal Messaging** - Rapid visual repetition for subconscious impact
- **Ken Ogger's Distance Operation** - Systematic compassion to unknown beings
- **Tibetan Buddhist Dedication** - Merit transfer to benefit others

### Core Concept

Each photograph represents a being in need. As the slideshow cycles through photos:
1. **Mantras repeat rapidly** (108+ times per photo) - like prayer wheel spinning
2. **Intentions flash subliminally** - positive messages absorbed
3. **Visual witness connection** - photograph links to actual being
4. **Automated dedication** - continuous blessing transmission
5. **Statistical tracking** - document benefit sent

## Technical Implementation

### Frontend Animation

**Mantra Pulsing** (10 Hz frequency):
```javascript
setInterval(() => {
  setMantraOpacity(prev => prev === 0 ? 1 : 0);
  incrementRepetitionCount();
}, 100); // 100ms = 10 Hz
```

**Why 10 Hz?**
- Subliminal threshold (faster than conscious processing)
- Alpha wave frequency (8-12 Hz, meditative state)
- Rapid enough for high repetition count
- Smooth visual effect

**Intention Cycling** (2 Hz frequency):
```javascript
setInterval(() => {
  cycleToNextIntention();
}, 500); // 500ms = 2 Hz
```

**Why 2 Hz?**
- Readable but rapid
- Keeps attention engaged
- Multiple exposures per photo
- Smooth rotation through all intentions

### Image Loading

Photos served via API endpoint:
```
GET /api/v1/blessing-slideshow/photo/{session_id}/{index}
```

Returns actual image file for display. This ensures:
- Server-side access control
- Consistent image serving
- No CORS issues
- Centralized management

Backend maintains session state:
- Current photo index

- Link to RNG session

Frontend polls for updates:
- Auto-advance timer
- Stats refresh

## Troubleshooting

### "No photos found in directory"
- Check directory path is absolute (/full/path/to/photos)
- Verify permissions (read access)
- Confirm image file extensions (.jpg, .png, etc.)
- Check for hidden files or system folders

### Slideshow won't start
- Ensure directory path is valid
- Select at least one intention
- Check mantra selection
- Try default settings first

### Images not displaying
- Check API server is running
- Verify photo files still exist
- Check browser console for errors
- Try refreshing page

### Mantra overlay not visible
- May be too fast to consciously see (this is intentional!)
- Check mantraOpacity state
- Verify animation timers running
- Look for flash/pulse effect

### High CPU usage
- Normal for rapid animations
- Reduce display duration
- Close other programs
- Use smaller images if possible

### RNG integration not working
- Create RNG session first
- Copy exact session ID
- Paste into slideshow config
- Check both services running
