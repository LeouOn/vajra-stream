1. STRATEGY
------------------------------------------------------------
Replaced the absolute `${API_BASE}` template form
(`${protocol}//${hostname}:8008/api/v1`) with proxy-relative URLs of
the form `/api/v1/...` across the frontend source tree. The Vite dev
proxy (`frontend/vite.config.ts`) already forwards `/api` to
`localhost:8008`. Production deployments route `/api/*` to the backend
via a reverse proxy (nginx/Caddy/Cloudflare) — the standard same-origin
pattern.

`apiUrl(path)` helper added to `frontend/src/utils/api.ts`:
    const baseUrl = import.meta.env?.VITE_API_BASE?.replace(/\/+$/, '') ?? '';
    export function apiUrl(path: string): string { ... }
Default: baseUrl === '' (proxy-relative).
Override: `VITE_API_BASE=https://api.example.com` at build time.

`API_BASE` kept as a deprecated backward-compatible alias
(`apiUrl('')` === '/api/v1') so any unmigrated call site continues to
resolve. New code MUST use `apiUrl('/foo')` or the `/api/v1/foo` literal.

WebSocket strategy UNCHANGED per ADR 004 (already proxy-relative at
`frontend/src/hooks/useWebSocketStable.ts`).
